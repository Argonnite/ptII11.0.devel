services/StockTick

Retrieve a stock price from a Yahoo server. This accessor reacts to a symbol input by issuing a query to a web server for the most recent trade prices of the common stock whose symbol is given by the input. When the server replies, this accessor produces the most recent trade price on the price output.

The request to the web server is asynchronous. This means that the outputs may not be produced in the same order as the inputs. FIXME: This is seriously problematic. Probably should change this to use a blocking call by default.

This accessor requires the optional 'httpClient' module, which may or may not be provided by an accessor host. Most hosts will provide this module.
